# Routing & Topology

One of Zenoh's defining features is **topology freedom** — the protocol imposes no constraints
on how nodes are arranged. This chapter explains how Zenoh routes data across arbitrary topologies.

## Chapters

- [Peer Mode](./peer-mode.md) — direct peer-to-peer; works without a router
- [Client Mode](./client-mode.md) — lightweight mode for constrained devices
- [Router Mode](./router-mode.md) — infrastructure node connecting segments
- [Multicast Scouting](./scouting.md) — automatic peer/router discovery
- [Regionalisation](./regionalisation.md) — hierarchical routing for massive scale
